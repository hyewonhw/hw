===테이블 구성===
행 : 로우(Row), 레코드(Record), 튜플(Tuple)
열 : 칼럼(Column), 필드(Field), 속성(Attribute)

===테이블 용어===
도메인 : 한 칼럼이 가질 수 있는 값의 범위(집합)
차수 : Degree, 한 테이블에 있는 칼럼의 개수
기수 : Cardinality, 한 테이블에 있는 로우의 개수

===슈퍼키===
유일성O,최소성X
테이블의 각 로우들을 식별할 수 있는 칼럼 또는 칼럼의 집합
고객 테이블을 분석하여 슈퍼키각 될 수 있는 칼럼이나 칼럼의 집합을 선정
판단기준 : 모든 로우가 서로 다른 값을 가질 수 있는가 ? -->유일성

===후보키===
유일성O,최소성O
최소한의 칼럼으로 구성된 슈퍼키만 선정한 것

===기본키===
유일성O, 최소성O, 개체 무결성(널값,중복된값 못가짐)
후보키중에서 DBA가 선택한 키를 의미
값이 변경되거나 의미가 변경될 수 있는 칼럼은 기본키로 부적절함
자연키 : 비즈니스 모델을 통해서 추출한 키, 즉 실제로 존재하는 데이터(이름, 키, 나이...)
인공키 : 실제로 존재하지 않는 데이터를 인위적으로 추가한 키, 주로 AUTO INCREMENT값을 사용함 (1번, 2번...)

DDL(Data Definition Language) - 데이터 정의어
- 데이터베이스 객체 추가/수정/삭제
- 작업 취소 불가
- 데이터베이스 객체(사용자, 테이블 등)
- 추가 : CREATE
- 수정 : ALTER
- 삭제 : DROP
